# 車両販売シート設計メモ

## 目的

営業用シートとは別に、車両販売の過程を管理する。

このシートで管理したいことは次の4つ。

1. どこから車を仕入れたか
2. 今誰に売ろうとしているか
3. その案件で実際にどんな資金移動が起きているか
4. 売上と利益がいつ確定したか

## シート構成

### 1. 案件マスタ

案件の基礎情報を持つ。

- 案件名
- 顧客名
- ブランド
- 車種
- 仕入元区分
- 仕入元名
- 売却先区分
- 売却先名
- 案件区分
- 進捗
- 在庫状態
- 利益計上ルール
- 残債精算あり
- 案件発生日
- 受注日
- 登録予定日
- 登録決定日
- 販売売上税込
- 販売売上税抜
- 粗利見込
- 粗利確定
- 紹介料売上
- 紹介料利益
- 関連案件
- メモ

### 2. 資金移動

現金の動きを記録する。売上かどうかはここで決めない。

- 発生日
- 資金区分
- 入出金
- 車両
- 案件
- 相手先区分
- 相手先名
- 査定額
- 仕入額
- 残債額
- 残債等支払額
- 資金移動額
- 予定日
- 収益対象
- 利益対象
- 関連案件
- 進行状況
- メモ

主な使い方:

- 顧客からの買取精算: `出金`
- ローン会社への残債一括返済: `出金`
- 顧客からの差額入金: `入金`
- 売上入金: `入金`

注意:

- ローン返済は `収益対象=対象外`
- ローン返済は `利益対象=対象外`
- 差額精算も、売上確定とは分けて扱う

### 3. 収益計上

売上と利益が確定したタイミングだけを記録する。

- 計上日
- 計上種別
- 売上区分
- 売上金額税抜
- 利益金額税抜
- 確定条件
- 連携先
- メモ

主な使い方:

- 通常の車両売上確定
- 粗利確定
- 紹介料計上
- 値引調整

### 4. 出金予定表

`資金移動` のうち `出金` だけを一覧化する。

### 5. 案件サマリ

案件ごとの全体像を見る。

- 仕入元
- 売却先
- 在庫状態
- 出金合計
- 入金合計
- 差引資金
- 確定売上税抜
- 確定利益税抜

## ルール

### 仕入元と売却先を分ける

同じ顧客案件でも、

- どこから仕入れた車か
- 今誰に売るのか

は別物として持つ。

### 資金移動と収益計上を分ける

現金が動いたからといって、売上や利益とは限らない。

例:

- ローン会社への返済
- 顧客との差額精算
- 一時的な立替

これらはまず `資金移動` に入れる。

### JLR のコンクエスト紹介料

コンクエスト案件で、紹介料が BRIDGE の売上かつ利益になる場合は、

- `資金移動` ではなく
- `収益計上`

に入れて `売上=利益` として扱う。

## 代表パターン

### パターンA: 顧客の残債付き下取から新しい車へ乗換

1. `案件マスタ` に案件作成
2. `資金移動` に顧客買取精算を登録
3. `資金移動` にローン会社への一括返済を登録
4. `資金移動` に顧客からの差額入金を登録
5. 納車後に `収益計上` で売上と利益を確定

### パターンB: コンクエスト紹介料のみ

1. `案件マスタ` に紹介案件を作成
2. 車両の仕入や出金は登録しない
3. `収益計上` に紹介料を登録
4. 売上金額税抜 = 利益金額税抜

## この設計で避けたいこと

- 営業用シート上で仕入元と売却先が混ざること
- ローン返済が売上に見えてしまうこと
- 差額精算が粗利に混ざること
- 紹介料案件と通常販売案件の管理方法が混ざること
