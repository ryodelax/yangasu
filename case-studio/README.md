# Case Studio — 案件 & 打ち合わせ管理アプリ

案件リストと打ち合わせの文字起こしを登録し、フィードバック・進捗を一元管理する Web アプリ。
ビルド不要の純粋な HTML/CSS/JS なので、**GitHub Pages にそのまま載せられます**。

## 機能
- **ダッシュボード** — 案件数・進行中・完了・文字起こし数のKPI、平均進捗、最近の活動フィード
- **案件リスト** — ステータス（リード/進行中/レビュー/完了/保留）・優先度・タグ・期限で管理、フィルタ＆検索
- **文字起こし** — 案件ごとに打ち合わせの書き起こしを保存（議事録の貼り付けOK）
- **フィードバック** — 良い点/要望/課題/メモを記入者付きで記録
- **進捗** — スライダーで達成率を更新、マイルストーンをタイムラインで可視化
- **ダーク / ライト** テーマ切替（設定は保存される）
- **書出 / 読込** — 全データを JSON でバックアップ・復元

## データの保存場所
ブラウザの `localStorage` に保存されます（このPC・このブラウザ内のみ）。
別端末と共有したくなったら、`js/store.js` の保存処理を Supabase 等の API に差し替えるだけで移行できます。

## ローカルで動かす
```bash
npx http-server case-studio -p 4173
# → http://localhost:4173
```
（単純な静的ファイルなので、index.html をブラウザで直接開いてもOK）

## GitHub Pages で公開する
1. このリポジトリを GitHub に push
2. リポジトリの Settings → Pages → Source を `main` ブランチに設定
3. 公開URLの末尾に `/case-studio/` を付けてアクセス

## 構成
```
case-studio/
├── index.html        画面の骨組み
├── css/styles.css    デザイン（テーマ・レイアウト）
└── js/
    ├── store.js      データ層（localStorage、後でAPI差し替え可）
    ├── ui.js         共通UI（モーダル・トースト・整形）
    └── app.js        画面描画とルーティング
```
