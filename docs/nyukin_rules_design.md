# 入金ルール設計メモ

## 目的

- `【最新】入金一覧` で案件ごとの見込みと実績の差分を管理する
- `キャッシュフロー表0515` へは、集約済みの見た目用シートではなく、案件粒度の見込みデータを渡す
- コンクエスト案件、保険会社立替案件、通常売掛案件を混同しない

## 現在の役割

- `振込入金リスト一覧`
  - `営業用` の元データから案件ベースで生成する一覧
- `銀行データチェック用`
  - 銀行実績の取込先
- `案件一覧 / コンクエスト管理 / 保険立替管理`
  - 実務確認用のビュー
- `売掛入金見込み管理`
  - 見た目確認用の集約表
- `売掛入金見込み明細`
  - CF反映と消し込み精度向上のための案件粒度データ

## 反映ルール

1. `案件一覧` で `入金済` の案件は見込みから除外する
2. コンクエストは `kind=conquest`
3. 保険会社立替は `kind=insurance`
4. 通常売掛は `kind=ar`
5. `売掛入金見込み明細` には以下を残す
   - 入金予定日
   - CFラベル
   - 入金予定額
   - 業務番号
   - 顧客番号
   - 請求先名
   - 予測キー

## CF側の使い方

- `キャッシュフロー表` は `売掛入金見込み明細` を優先して読む
- 銀行実績に `業務No` または `顧客No` がある場合、CF上の見込みノート内の同一キーを優先して消し込む
- キーが無い場合のみ、従来どおり名前・金額・日付近接で照合する

## 今後の改修候補

- コンクエスト月次一括入金を複数案件へどう按分するかのルール化
- 保険案件の差額入金を `一部入金` として明示管理する補助列の追加
- `【最新】入金一覧` のメニューから `売掛入金見込み明細` 更新と `CF反映` を一気通貫で呼ぶ導線追加
