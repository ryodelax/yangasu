### 2026-05-29: 引き継ぎ情報をコードと分離して管理する

- 背景:
  ブリッジ案件の関連ファイルが散在しており、会話だけでは再開文脈が失われやすい。
- 決定:
  `docs/bridge-handover/` を案件引き継ぎの基準フォルダとして使う。
- 理由:
  AI再開時にも人への引き継ぎ時にも同じ資料を使えるため。
- 影響範囲:
  案件概要、未完了タスク、判断経緯の保存方法。
- 保留事項:
  将来的に案件専用リポジトリへ移すかは未決定。

### 2026-05-29: CF表への入口は main 側メニューではなく CFブック自体にも持たせる

- 背景:
  main 側の `運用` メニューからは CF反映を実行できるが、ユーザー要望は `キャッシュフロー表を開いた状態で直接押せる入口` だった。
- 決定:
  `cashflow_integration_v5.gs` に CFブック専用メニュー `CF反映 > ① CF反映を実行` を持たせ、installable `onOpen` trigger で表示する。
- 理由:
  main ブックに戻らずに運用でき、次回以降も同じ導線を維持しやすい。
- 影響範囲:
  `onOpen()`、`buildCashflowOperationsMenu_()`、`installCashflowWorkflowOpenTrigger_()`、`setupCashflowSystem()`、および main 側の初期化導線。
- 保留事項:
  live 環境では trigger 作成が未完の可能性があり、Apps Script 実行権限と `clasp` 再認証状態を要確認。
