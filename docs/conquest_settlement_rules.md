# コンクエスト相殺ルール

## 基本方針

- キャッシュフロー表には、コンクエスト関連の予定を明細単位で出す
- ただし実際の金銭授受は月次の相殺で行う
- そのため `見込み明細` と `月次相殺実績` は別物として扱う

## 見込み明細の分類

### 1. 定期支払い

- `マセラティCS6810分割`
- `ディフェンダー デモカー（466712）`
- `ブリッジ1,600万円返済`

これらは案件単発ではなく、定期支払いマスタから月次で起こす。

### 2. 粗利税込請求

- `マセラティ3月分` のような項目
- 元データは `管理用` シート上のマセラティ粗利
- 金額は `粗利 × 1.1`

### 3. 経費請求

- `経費3月分`
- 元データは `ブリッジ -> コンクエスト` の経費請求書
- 添付PDF例では `-83,000`
- 毎月ほぼ固定で発生する場合は、請求書発行前でも `見込み` として先行計上してよい
- 請求書発行後は実額で上書きする

### 4. JLR顧客別明細

- `山本晋平様 DF`
- `吉野幸美様 F-PACE`
- `小林弘治様 RRE.314702`

JLR系は顧客別の取引として個別明細で持つ。

### 5. 下取車案件

- `下取車DF`
- `下取車RRE`

これは `コンクエスト -> ブリッジ` の請求書が存在する案件。
通常の顧客売掛ではなく、コンクエスト仕入れ系の明細として扱う。

### 6. 請求先集計

- `JLR請求書営業部門3月分`
  - `請求先名 = 株式会社　コンクエスト　ｼﾞｬｶﾞｰﾗﾝﾄﾞﾛｰﾊﾞｰ営業部門`
- `JLR請求書3月分`
  - `請求先名 = 株式会社コンクエスト　ジャガーランドローバー広島`

これらは営業用データから請求先名で集計して作る。

### 7. 紹介料

- `紹介料 小林弘治 RRE`
- 元データは `営業用 / 顧客対応状況（車両販売）`
- 金額は `確定利益 × 1.1`
- 月の基準日は `登録決定日`

紹介料関連として別枠管理する。

## CFへの出し方

- 明細はそれぞれ個別の予定行としてCFへ書く
- 収支方向は実態に合わせる
  - ブリッジが受け取る見込みは入金予定
  - ブリッジが支払う見込みは出金予定
- 月末には `コンクエスト相殺 YYYY/MM` の集計結果を別管理する

## 実績消し込み

- 銀行実績は月次相殺の総額で入る前提
- そのため実績は明細1件ずつに1対1で当てない
- 実績入金時はまず `コンクエスト相殺` グループへ当てる
- 同月グループの明細には `相殺済` 状態を付ける
- 実績確定段階では `コンクエスト相殺一覧明細` を作成し、月内明細と相殺実績の対応を残す
- CF上では
  - 明細予定は内訳として保持
  - 実際の現金移動は `相殺実績` として別行で表現

## 必要なデータキー

- `settlement_group`
  - 例: `CONQUEST:2026-04`
- `detail_kind`
  - `recurring`
  - `gross_invoice`
  - `expense_invoice`
  - `jlr_detail`
  - `tradein_purchase`
  - `sales_dept_invoice`
  - `jlr_invoice`
  - `referral_fee`
- `biz_no`
- `cust_no`
- `bill_to`
- `car_name`

## 実装上の含意

- 今の `銀行実績 1件 -> CF見込み 1件` 消し込みでは足りない
- コンクエストだけは `月次相殺グループ` 単位の消し込みロジックが必要
- `売掛入金見込み明細` はその前段データとして使う
